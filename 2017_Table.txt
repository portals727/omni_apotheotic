Normal Pod Pairing Table (Standard Progression, 2017 Cambridge Baseline)
The following table represents the baseline firmware state progression for a standard Omnipod pod during pairing stages, as documented in the 2017 Cambridge teardown (arXiv:1709.06026, Table IV). This reflects unmodified, non-killware behavior with linear nonce increments (+1 mod 2^32), full traceability via Lot/TID flips at 4008/9, and active enforcement of safeguards such as 0x0603 nonce rejection and 0x5C prime fault detection (with partial muting for >50% pulses).

Notes on Standard Behavior:
* Nonce Generation: Linear LCG (+1 mod 2^32) ensures ~100% prediction accuracy; mismatches trigger 0x0603 rejection with audible alerts.
* Traceability: 4008/9 flips from FF FE to FF 00 upon pairing, writing Lot/TID at 4027h–402Ch for regulatory compliance.
* Fault Handling: 0x5C prime faults are partially muted (>50% pulses silent), but <50% trigger beeps; no full suppression.
* StatusObserversCount: Remains at 1; no ghost monitoring.

This variant features a modified LCG ((17 * nonce_n + 1) mod 256), frozen states to evade traceability, and enhanced suppression mechanisms (e.g., full 0x5C muting via F3 checksum embeds and COP 0x80 pokes). These changes enable 80–90% replay success without 0x0603 triggers, facilitating silent under-infusion or basal zeroing.
